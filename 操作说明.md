## 一、 👥 角色划分与双重权限模型 (RBAC & ACL)

为了保障词条版本定稿时的绝对权威性，系统设计了「全局账号角色」与「项目内成员角色」的双重权限控制，防止译员误删或误改已审定的词条。

### 1. 系统级角色 (System Roles)
在 Supabase 数据库的 `users` 表中分配：
*   **管理员 (`admin`)**：系统超级管理员。拥有系统的全权管理权限，包括：
    1.  对词条进行 **“批量审批 (Batch Approve / Reject)”** 并改变工作流状态。
    2.  对词条行进行 **“防冲突锁定 (Lock / Unlock)”**。
    3.  系统全局设置中保存/修改 Dify API 工作流密钥。
    4.  发起历史快照的一键还原回退（回退后状态自动变更为 `APPROVED` 免审）。
*   **普通用户 (`user`)**：前端普通协同开发或翻译官。仅可进行日常词条翻译录入，无权进行行锁定、批量审批以及全局密钥配置；发起快照回退时，修改后的词条将降级流转为 `PENDING_REVIEW` 待审状态。

### 2. 项目级角色 (Project Roles)
在 `project_members` 关联表中分配，针对特定项目起效：
*   **所有者 (`owner`) / 编辑者 (`editor`)**：可对该项目下的词条进行日常录入、提交翻译、版本创建。
*   **查看者 (`viewer`)**：仅只读查看、检索、比对以及导出 CSV，**所有写操作按钮在 UI 界面上会自动置灰或隐藏**（适合分发给外协测试团队核对词条）。

## 二、 🚀 核心升级功能操作指引 (v0.3)

### 1. 行级防冲突锁定 (Row Locking)
*   **场景**：临近版本发包前，为了防止译员或开发人员意外修改已定稿的词条，管理员可将行锁定。
*   **操作**：
    *   管理员在表格右侧的操作列中点击 **“小锁 (Lock)”** 图标，该行即被物理锁定。
    *   锁定后，非管理员用户的界面上修改按钮会变灰，悬浮提示 `未解锁/无权修改`，后端 API 在接收到 PUT 修改请求时会抛出 `FORBIDDEN` 强制拦截。

### 2. 翻译来源标记 (Translation Telemetry Badges)
*   **定义**：在多语言翻译格子下，悬浮或右侧会有明确的**来源角标 (Badge)**，用以追溯翻译质量：
    *   `ai` (机器人图标)：由 Dify 大模型预翻译生成。
    *   `tm` (记忆库图标)：由翻译记忆库 (Translation Memory) 匹配得出。
    *   `human` (绿色打勾图标)：经由人工（开发人员或翻译官）手动输入修改确认。
*   **流转**：当人工手动双击修改了某个 AI 生成的翻译并保存后，该字段的来源会自动从 `ai` 流转升级为 `human`，实现质量评级的闭环。

### 3. AI 用量 Token 折线大盘 (Telemetry Dashboard)
*   **定义**：在“仪表盘看板”顶部，新增了 AI 翻译大盘。
*   **图表展示**：
    1.  **AI 总 Token 消耗**：清晰统计当前项目所使用的累计大模型 Tokens 数。
    2.  **7 天用量折线趋势图**：使用高颜值 SVG/Canvas 动态绘制过去 7 天内 Tokens 消耗与翻译耗时（秒）的走势，方便 PM 监控翻譯成本。

### 4. 语种就绪进度矩阵 (Progress Metric)
*   **定义**：在仪表盘右侧，以横向红、黄、绿三色进度条直观展示 **16 个目标语种的就绪覆盖率**。
*   *就绪率 = 该语种已有翻译的词条数 / 总词条数。* 页面一旦打开，哪个语种还缺翻译、缺多少一目了然，方便发包前做精准补漏。

### 5. 版本比对与一键同步 (Version Sync)
*   **版本比对**：在“版本比对”Tab 中设定源版本 A 与目标版本 B，点击比对，系统会自动剔除弯直引号、省略号等假差异，输出最精准的 `新增`、`删除`、`修改` 清单。
*   **一键同步回飞书 (Sync Back)**：在飞书小组件（插件模式）下，比对出差异后，可以点击右上角 **“同步回飞书”**，系统会直接把差异的字段增量改写回飞书云端的多维表格单元格中，实现云端数据的物理对齐。

## 三、 典型业务协作工作流推荐

### 💡 迈金多人协同标准工作流 (发版四步法)

```
开发录入(DRAFT) ➔ AI预翻译(PENDING) ➔ PM/翻译审核(APPROVED) ➔ 锁定行并同步(PUBLISHED)
```

1.  **词条录入 (开发阶段)**：
    *   开发人员（全局 `user` 角色）在独立浏览器网页或飞书组件里，录入新页面的中文词条和 `KW`。此时词条状态默认为 `DRAFT` (草稿)。
2.  **AI 智能翻译 (预翻译)**：
    *   译员或开发选中这几行，点击 **“批量 AI 翻译”**。Dify 翻译工作流运行，扫完所有空缺的多语言，自动打上 `ai` 角标，词条状态流转为 `PENDING_REVIEW` (待审核)。
3.  **人工审核 (定稿阶段)**：
    *   项目负责人（全局 `admin` 或项目 `owner` 角色）双击有异议的词条进行手动微调（角标自动升级为 `human`），并点击 **“批量审核通过”**，词条状态变更为 `APPROVED` (已审批)。
4.  **行锁定与发包同步**：
    *   项目负责人勾选已审定的词条，点击 **“锁定行”**。
    *   点击 **“一键同步”**，将云端数据库中的最终 APPROVED 数据全量更新对齐回飞书云端表格，直接导出作为固件编译的词条配置文件。

## 四、 常见问题解答与故障排查 (FAQ)

#### Q1: 登录时提示 `⚠️ 网络连接失败: Unexpected token 'T', "The page c"... is not valid JSON`？
*   **原因**：前端把接口请求发错了地方（发到了 Vercel 静态服务器上，从而返回了 Vercel 404 HTML）。
*   **排查步骤**：
    1.  请检查您在 Vercel 控制台的项目环境变量中，**`VITE_API_BASE_URL`** 环境变量是否配置正确（其值应当是您的 Render 后端地址：`https://glossa-hub.onrender.com`，且**结尾千万不能带 `/`**）。
    2.  在 Vercel 环境变量修改后，**必须手动在 Vercel 页面点击 Deployments -> Redeploy (重新部署)**，否则新的变量不会打包生效。
    3.  若报错尾部的当前请求地址正确，说明您的网络请求在公网被**本地网络代理工具（如 Clash、Surge 等）**拦截了。请暂时关闭代理软件或切换为“直连模式”，强制刷新页面后再试。

#### Q2: 为什么在 Render 日志中看到 `PostgreSQL 连接失败，自动切换为 SQLite`？
*   **原因**：在 Node `pg` 驱动中连接 Supabase 时，如果没有提供 SNI servername 会被 Supabase 防火墙拦截。
*   **解决**：我们已在 v0.3 中更新了 `server.cjs` 的 SNI 主机名自动提取补丁，系统现在会自动提取并在握手时发送 `servername`。请确保您的 `DATABASE_URL` 在 Render 中配置完整且密码替换正确即可。

## 五、 📝 词条多语言文案与格式化规范

为了保证多语言翻译在代码开发时的拼接正确性，GlossaHub 采用经典的 **C 语言 `printf` 格式化字符串标准（Format Specifier）** 处理变量占位符。

如果在文案中遇到动态变动的内容（如数字、设备名等），**严禁使用真实的文字或自定义括号（如 `{num}`）占位**，请统一使用以下标准占位符：

### 常用占位符说明
*   **`%d` (整数数字)**：用于动态展示具体的整数。例如：`已骑行%d条记录`（代码渲染时会变为：`已骑行5条记录`）。
*   **`%s` (文本字符串)**：用于动态展示具体的文本名称。例如：`确定切换至【%s】？`（代码渲染时会变为：`确定切换至【夜间模式】？`）。
*   **`%%` (百分号本身)**：如果文案里确实需要显示 `%` 符号，为了防止和变量前缀冲突，必须连写两个 `%` 进行转义。例如：`已完成%d%%`（代码渲染时会变为：`已完成50%`）。
*   **`%02d` (补零格式化)**：用于要求数字必须至少显示两位，不够则在前面补零。例如：`v%d.%d%02d`（代码渲染时可能是版本号拼接：`v1.0.03`）。
*   **`\n` (换行符)**：表示文本在此处强制换行（Newline）。因为 CSV 导出时真实的换行容易破坏表格结构，在长文案（如弹窗说明）中需要换行时，直接在文本中写 `\n`。例如：`第一行\n第二行`。

### 给翻译员/AI 的建议
不同语言的语序可能完全不同（例如：“把 `%s` 移出了 `%s`” 在英文中可能是 “Removed `%s` from `%s`”）。
使用 `%d`、`%s` 可以让翻译员在不同语言下**自由调整变量在句子中的位置**，这也是系统推荐的标准最佳实践！如果担心机器翻译丢失占位符，可以在词条的**备注释义**列中注明：*“含变量 `%d`，请原样保留占位符”*。

### 标点符号规范（全角与半角）
在多语言本地化（L10n）中，标点符号的使用必须严格遵循目标语言的排版习惯：
1. **中文及东亚语种（中、日、韩）**：统一使用**全角标点符号**（`，` `。` `！` `？` `：“”`）。
2. **英文及其他西方语种（法、德、西等）**：统一使用**半角标点符号**（`, ` `. ` `! ` `? `），并且**标点符号后必须紧跟一个半角空格**（除非是句末）。
3. **中英文混排**：在中文词条中如果夹杂了英文单词或数字（如 `App`、`100`），英文字母和数字本身必须是**半角**，但句子中的标点符号仍要保持**全角**（例如：`请连接 App，然后重试。`，这里逗号和句号都是全角）。
