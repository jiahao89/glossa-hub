# GlossaHub - 迈金码表词条协同与版本管理平台 产品需求文档 (PRD)

## 1. 项目背景与业务痛点
迈金（Magene）旗下拥有多款面向全球销售的智能 GPS 骑行码表。码表系统内置支持 15+ 种语言包，所有显示的文字词条数量庞大，目前存在以下管理痛点：
1. **人工录入易出错**：历史使用电子表格或传统文档维护词条，增删改查纯手工修改，容易导致行错位、漏译或格式损坏。
2. **多语种翻译成本高**：新词条录入需多国语言翻译，人工跟进翻译周期长、协作繁琐。
3. **词条版本管理混乱**：词条根据固件版本迭代，旧版是以独立表格隔离，缺乏版本间的清晰差分对比（Diff），难以追溯“谁在什么时间修改了什么词条的哪种语言”。
4. **多人协同效率低**：本地维护导致文件版本冲突，难以做到多人同时在线编辑、审核和实时同步。

为了解决以上痛点，本项目已建设一个**独立的 Web 端多人协同翻译平台 GlossaHub**。系统脱离第三方低代码表格环境，基于自研前后端服务，支持多人在线同时维护，并提供自动化的 AI 翻译流水线和严谨的版本差分比对能力。

---

## 2. 产品定位与建设目标
*   **产品定位**：一个独立的 PC 端大屏多人协同翻译与固件版本管理 Web 平台。它是一个基于现代前端框架（React）与自研后端服务（Vite + Express + SQLite/PostgreSQL）构建的 SPA 应用。
*   **核心协作机制**：**云端多人协同与安全鉴权隔离**。系统通过后端统一数据库存储数据，采用 JWT 鉴权隔离与 bcrypt 慢哈希加盐存储用户密码，保障系统运行与团队协同安全。
*   **核心翻译引擎**：**Dify 工作流 (Workflow) 后端安全代理**。前端触发 AI 翻译，请求由后端网关拦截并代理发送给 Dify API（不向前端公开 API Key），从而保护核心商业资产的安全。
*   **核心建设目标**：
    *   **项目与版本多级管理**：支持按设备/固件创建不同的“项目”（Projects），并在项目下管理“固件版本数据表”（Versions）。支持表重命名与数据一键克隆。
    *   **宽屏大表格视图**：提供超宽的多语种对照数据网格，方便用户横向滚动浏览全部 15+ 种语言翻译，支持固定核心列。
    *   **一键批量 AI 翻译**：用户录入中文后，通过 Dify 代理接口一键自动生成 15+ 种语言的翻译。
    *   **动态专业词汇库**：支持动态自适应多列 CSV 导入，可按 CSV 结构自动匹配表头渲染，避免翻译名词打架。
    *   **固件版本比对（Diff）**：自动读取不同版本，智能对比相邻版本的“新增 (ADD)”、“删除 (DEL)”与“修改 (MOD)”变动。
    *   **安全加固**：包含登录频率限制（Rate Limit）、JWT 密钥环境变量、统一异常格式化，严防敏感数据与错误泄露。

## 3. 功能模块详细说明

### 3.1 仪表盘看板 (Dashboard)
*   **全局数据统计**：展示固件版本总数、中文词条总键数、完全翻译完成词条数，以及全语种格子翻译覆盖率。
*   **进度进度卡片**：列出当前项目下的所有版本数据表，计算并显示每个表的翻译就绪百分比、包含词条数、以及完全翻译就绪词条数。
*   **变更记录 (CST 东八区)**：显示最近 5 条的修改动态（包含操作人、改动表名、KW 及具体细节）。
    *   *时间显示要求*：强制使用东八区（CST）进行对比。如果修改发生在今天，仅显示具体时分秒；非今天修改的事件，则显示 `YYYY-MM-DD HH:mm:ss` 年月日与时间。

### 3.2 词条协同网格 (Translation Manager)
*   **主对照表格**：列头固定显示“所在页面”、“字号类别”、“KW（键名）”和“中文（zh_cn）”，以及所有启用的目标语言。
*   **行级与单元格级编辑**：支持直接双击修改单元格，红色高亮表示改动尚未保存。
*   **自动保存与防抖**：采用 1.5 秒防抖（Debounce）合并保存机制，用户停止输入后自动向后端更新数据，避免频繁触发 HTTP 请求。
*   **AI 批量自动翻译**：支持“翻译当前行”和“翻译全部未翻译行”。请求由后端代理，通过 Dify 工作流多线程翻译并回填。

### 3.3 专业词汇库 (Glossary Dynamic Engine)
*   **通用动态多列 CSV 导入**：支持导入包含任意语种与字段的多列 CSV。系统自动解析 CSV 标题栏（精准匹配 `CN`、`EN` 等翻译字段，排除 `component` 等干扰），并在数据库中保留全量字段。
*   **表头自适应渲染**：表格不使用硬编码列名，而是根据导入词汇表的实际 `headers` 动态构建 Th。非自定义词汇表则自适应回退到默认的“中文专业术语、英文翻译对应、说明/定义”三列显示。
*   **表内数量展示**：词汇表下拉菜单旁，实时常态展示当前词汇表内包含的词条数量（`共包含词条：X 条`）。

### 3.4 版本变更对比 (Comparison)
*   **双版本对比**：允许选择“对比版本 A”和“基准版本 B”。
*   **三类差异分析**：
  *   **新增**：A 独有的词条（绿色高亮）。
  *   **删除**：B 独有的词条（红色高亮）。
  *   **修改**：同一 KW 下，A 和 B 翻译存在差异的词条（黄色高亮，并对比 old 和 new 翻译值）。
*   **假差异过滤 (Normalization)**：自动清洗和标准化省略号、引号、空格差异，排除非内容实质变动的干扰。

### 3.5 数据表管理 (Versions Manager)
*   **大表一览**：显示当前项目下的所有固件大表，包括表名、创建时间、最近修改时间、创建人。
*   **重命名表名**：支持在行操作中点击“编辑”按钮，弹窗输入新名称更新数据表名字。
*   **数据克隆**：支持一键将某大表的所有词条数据复制克隆到另一个表，确保版本迁移时的初始数据对齐。
