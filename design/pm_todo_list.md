# GlossaHub - 调试与配置 PM 检查清单 (PM TODO List)

为了确保 GlossaHub 插件能与您的飞书多维表格及 Dify 工作流 100% 完美对接，请按照以下清单逐步确认与测试。

---

## 1. Dify 工作流端检查 (Dify Checklist)

由于 Dify API 是通过 **“接口地址 + 密钥 (API Key)”** 来识别具体工作流的，只要密钥正确，Dify 就会自动执行对应的项目。

*   `[ ]` **发布最新版本**：请确认您的 Dify 工作流在修改完变量后，已在右上角点击了 **“发布”** ➜ **“更新运行”**。如果不发布，外部 API 将一直调用旧版工作流。
*   `[ ]` **检查接口地址 (Base URL)**：
    *   如果您使用的是 Dify 官方云服务，接口地址为 `https://api.dify.ai/v1`（代码中已默认内置此地址）。
    *   如果是企业私有化部署，请在小组件的“引擎设置”中修改为您的私有部署 API 域名（例如：`http://dify.magene.com/v1`）。
*   `[ ]` **开始节点变量确认**：请确认您的 Dify “开始节点”中的 4 个必填变量名称（大小写与字符）与下方完全一致：
    1.  `KW` (单行文本)
    2.  `text` (段落文本，代表中文原词)
    3.  `所在页面` (段落文本)
    4.  `语种` (段落文本)
*   `[ ]` **关联知识库**：确保“知识检索”节点已经关联了您的 Magene 专业词汇字典，并且“查询变量”绑定的是 `开始.text`。

---

## 2. 飞书多维表格数据表结构检查 (Bitable Column Checklist)

插件读取和回写数据完全依赖列名匹配，请确认多维表格（如 `3.1`、`3.2` 工作表）中的**列标题名称**与下方完全一致（字错一个都会导致无法读取或写入）：

*   `[ ]` **核心控制列**：
    *   **`KW`**（文本类型，用于标识唯一词条键值）
    *   **`中文`**（文本类型，源中文词汇）
    *   **`所在页面`**（文本类型，上下文限制，必须为这个名字）
    *   **`负责人`**（文本类型，开发或 PM 的名字）
*   `[ ]` **19 个目标语种列**（全部为文本类型，名字必须完全一致）：
    *   `英文`、`法语`、`德语`、`西班牙语`、`意大利语`、`葡萄牙语`
    *   `韩语`、`日语`、`俄语`、`波兰语`、`繁体中文`、`丹麦语`
    *   `捷克语`、`瑞典语`、`挪威语`、`荷兰语`、`泰语`、`芬兰语`、`土耳其语`
*   `[ ]` **版本表名称**：进行版本对比时，工作表名称必须为数字（如 `3.1`、`3.2` 等，可带 `v` 或 `V` 前缀如 `v4.0`），否则插件无法自动识别为可对比的版本。

---

## 3. 本地插件调试步骤 (Local Debugging Steps)

*   `[ ]` **开发服务确认**：确认本地开发服务已经启动（我已经在后台为您运行了 `npm run dev`）。本地服务地址通常为 `http://localhost:5173`。
*   `[ ]` **飞书加载小组件**：
    1.  打开您的测试多维表格，点击右上角 **“小组件”** 侧边栏。
    2.  点击 **“添加小组件”** ➜ **“创建小组件”** ➜ 命名为 `GlossaHub 调试版`。
    3.  开发模式选择 **“本地开发 (Localhost)”**。
    4.  地址栏填入：`http://localhost:5173` 并保存。
*   `[ ]` **测试引擎连接**：
    1.  在飞书加载出来的插件中，点击 **“引擎设置”** 页签。
    2.  此时密钥应该已经自动填入 ``。
    3.  点击 **“测试 API 连接”**。
    4.  *预期结果*：提示“连接成功”，且底部状态灯亮起绿色。如果失败，请在此页面的控制台中查看报错。
*   `[ ]` **单条/批量翻译调试**：
    1.  在主工作区选择一条中文未翻译的词条，双击打开编辑框。
    2.  点击中文旁边的 **“AI 预翻译”**，观察 19 个语言输入框是否被瞬间填满。
    3.  点击保存，观察多维表格对应的列是否自动写入了翻译结果。
    4.  测试 **“批量翻译”** 功能，确保弹窗预览正常、修改预览值后一键写入表格正常。
*   `[ ]` **CSV 导入导出与版本对比**：
    1.  点击 **“导出”**，检查下载下来的 CSV 表格，在 Excel 中打开是否有乱码。
    2.  在表格里新建一张工作表命名为 `3.2`，在老表命名为 `3.1`，切换到 **“版本对比”** 页签运行对比，检查高亮和 Diff 显示是否符合 PRD 要求。
