一、 全局角色权限与交互模拟

为了防止词条在临近版本封包定稿时被译员或开发意外改动，系统采用了双重权限防护机制：

  • 1

    系统级全局角色 (users.role)

    在库中为账号配置全局 admin（超级管理员）或 user（普通译员），控制锁定、审批以及保存 Dify Key 等敏感接口。

  • 2

    项目级成员角色 (project_members.role)

    针对具体产品项目隔离。分为 owner（项目所有者）、editor（编辑者）和 viewer（只读查看，UI 操作全部变灰）。

二、 v1.0 版本升级核心新功能

在多人协同发包的复杂业务场景下，平台提供了八大核心功能来保障数据质量与提效：

  • 1

    🔒 行级防冲突锁定 (Row Locking)

    管理员（Admin）或词条 Owner 可以在词条最左侧状态栏直接点击 🔓/🔒 图标进行加锁与解锁。上锁后，该行记录将受物理防护：双击打开时提示只读警告且输入框与 AI 翻译按钮全部禁用变灰；在批量翻译、大表数据导入更新、批量分类设置等高敏感操作中也会被自动安全跳过，防止冲突覆写。

  • 2

    🏷️ 翻译来源角标 (Telemetry Badges)

    每个语种的单元格右侧均带有明确的来源标识：ai (智能算法生成)、tm (历史记忆库复用)、human (人工校验并定稿)。一旦人员对译文进行二次微调保存，来源角标会自动无缝升级为最高可信度的 human。

  • 3

    📈 AI 用量 Token 大盘 (Analytics)

    仪表盘顶部引入了消耗统计大盘，自动追踪每次翻译的 Token 损耗与用时，并绘制最近 7 天的每日消耗趋势折线图，协助监控与核算 AI 的成本支出。

  • 4

    📊 语种就绪进度矩阵 (Progress Metric)

    主面板配有直观的语种覆盖进度矩阵，支持对 16 个出海语种进行细粒度分析，实时标红/黄/绿并计算百分比，使得未译缺口一清二楚。

  • 5

    🔄 比对差异一键写回飞书 (Sync Back)

    提供强大的跨表/跨版本“多维表格级”差异对比工具，能够过滤省略号、双引号等格式假差异。支持筛选后一键差异写回，完成飞书数据源更新。

  • 6

    🧠 跨版本翻译智能参考 (Cross-version Reference)

    在编辑词条时，系统会基于当前词条的唯一主键 `KW`，自动检索当前项目内除当前数据大表外其他所有版本的数据大表。如果该 `KW` 在其他版本中已存在译文，右侧将把对应的大表名称及翻译结果调出来供译员对比与一键复制，保障迈金跨版本产品的翻译一致性。

  • 7

    ⏱️ 词条修改历史追溯与一键回退 (Audit Logs & Rollback)

    编辑弹窗的右侧“修改历史”时间轴，会完整记录该词条的一生：谁在何时把何种语种从什么值改成了什么值。点击任意历史卡片底部的“回退”，即可将当前词条的整体语种状态一键瞬间恢复到该历史时间点。

  • 8

    📂 场景化语境与多维元数据绑定 (Context & Tags)

    词条不仅包含 `KW`，还完整支持了“所在页面（如个人中心配对页）”、“字号类别”以及上下文“Context”的写入，译员可结合表头的高级过滤，轻松对特定应用模块的文案进行批量翻译与归档。

三. 典型协同业务工作流推荐

推荐迈金多人高效协作的 **“四步发版工作流”**：

开发录入 (DRAFT) ➔ AI预翻译 (PENDING) ➔ 人工校验与审批 (APPROVED) ➔ 锁定行并同步回飞书 (PUBLISHED)
  1. A

    开发人员：词条录入

    开发人员（全局 `user` 角色）在网页端或飞书组件里快速建立新页面词条，录入 `KW` 和 `CN(中文)`。此时词条状态默认为 `DRAFT` (草稿)。

  2. B

    翻译团队/大模型：AI 智能翻译

    选中新增行，一键点击批量 AI 预翻译。系统自动填入 16 个语种翻译，打上 `ai` 来源角标。词条状态流转为 `PENDING_REVIEW` (待审核)。

  3. C

    项目负责人/译员：人工定稿与审批

    对 AI 翻译不满意的字词进行双击编辑微调（微标自动更新为 `human`），确定合格后点击 **“批量审批通过”**。词条状态升级为 `APPROVED` (已审批)。

  4. D

    项目负责人：一键锁定与同步

    将已定稿的行勾选并 **“锁定”**（防改）。点击 **“一键同步”**，把本地数据库所有已审批的词条全量同步并更新到飞书云端表格，完成出海发包。

四、 常见问题与故障诊断 FAQ

在部署与使用过程中，如遇到以下常见错误，请参考对应排查步骤解决：

Q1: 登录报错 `Unexpected token 'T', "The page c"... is not valid JSON`？ ▼

原因：前端请求没有被送往后端（请求了 Vercel 的静态服务，返回了以 `The page could not be found` 开头的 404 HTML 页）。

解决步骤：

  1. 登录 Vercel 项目页面 ➔ Settings ➔ Environment Variables。
  2. 检查是否正确添加了 **`VITE_API_BASE_URL`**，值必须是您的 Render 后端 API 域名：`https://glossa-hub.onrender.com` (注意：末尾绝对不能带有斜杠 `/`)。
  3. 添加或修改后，必须进入 Vercel 的 Deployments 菜单，对最新那次部署选择 Redeploy (重新部署)，否则新环境变量不会被打包进静态文件。
  4. 如环境变量无误，说明您的请求在公网遭到了**本地科学上网/网络代理工具（如 Clash、Surge 等）**的拦截（导致 90ms 瞬间本地返回错误页）。请暂时关闭代理或切换为“直连模式”，强制刷新页面即可解决。
Q2: 环境变量 `CORS_ORIGINS` 怎么配置？结尾需要斜杠吗？ ▼

原因：后端 CORS 跨域校验是严格的字符串比对。浏览器在跨域发送请求时携带的 `Origin` 头是 **`https://glossa-hub.vercel.app` (永远不带斜杠)**。

解决步骤：

  1. 登录 Render.com 控制台 ➔ 进入您的后端 API 选项卡。
  2. 检查 `CORS_ORIGINS` 环境变量值。如果是带有 `/`（如 `https://glossa-hub.vercel.app/`），请点击 Edit，将末尾的斜杠物理删掉。
  3. 点击 Save 储存，Render 自动热重启完毕后即放行跨域。
Q3: 在 Render 日志中看到 `no tenant identifier provided (external_id or sni_hostname required)` 自动降级为 SQLite 是什么情况？ ▼

原因：这是 Supabase 实例的 SSL SNI 校验机制限制，如果连接池在 TLS 握手时没收到 servername，会拒绝外网握手连接。

状态与解决：我们已在 v1.0 的 `server.cjs` 代码中部署了 SNI 自动提取与填充的主机名补丁。现在 Render 重新发布后，系统能够自动建立安全的 SNI 通道，您不需要手动执行任何代码修改。

Q4: 数据库密码中含有 `@` 或者是特殊保留字符，导致解析失败、无法连接？ ▼

原因：PostgreSQL 标准 URI 连接串以第一个 `@` 符号作为密码和主机名（Host）的分隔符，若密码本身带有 `@` 等字符，常规的 URL 解析库会发生字段错配，从而导致密码被硬性截断并报错。

解决与状态：我们已在 `server.cjs` 后端解析层引入了贪婪正则切分与 URL 安全解码机制。系统会从右向左锁定最后一个 `@` 分隔符，自适应还原特殊字符，因此现在无论您的密码是否包含 `@`，系统在 Render 重新启动后都能成功验证通过。

Q5: 本地测试 Dify 翻译正常，但部署上线后总是报错提示“Dify 引擎未配置”？ ▼

原因：本地 SQLite 数据库将 `dify_config` 字段作为 TEXT（字符串）形式返回，后端通过 `JSON.parse` 进行解析；而线上 Supabase PostgreSQL 中由于该列为 `jsonb` 类型，pg 驱动会直接返回已解析好的 JS 对象。如果对 Object 强行执行 `JSON.parse` 会触发 JS 语法异常并被 catch 捕获，造成配置为空的报错假象。

解决与状态：后端中转 API 已经针对 `project.dify_config` 部署了类型自适应逻辑。如果检测到其已为对象则直接读取，否则再执行 `JSON.parse` 兼容文本，目前本地 SQLite 和云端 PostgreSQL 的 Dify 翻译皆可畅通运行。

Q6: 免费的 Supabase 云数据库因为 IPv6 限制在 Render 容器中报 `ENETUNREACH` 错误？ ▼

原因：Supabase 的直连地址仅发布了 IPv6 记录，而 Render 等静态云容器目前仅支持 IPv4 出网，导致物理网络无法送达。

解决与状态：本系统已强制挂载了 IPv4 DNS 优先级解析补丁。同时为了确保可用，代码在连接时会自动将直连地址在内存中重映射为您项目专属的官方池化域名：`aws-1-ap-northeast-2.pooler.supabase.com:6543`，完美绕过网络限制并接通。
